# 2026-03-19

## mm_v7 HGEMM 实现
- 在 `matmul/mm_v7.cu` 中实现了高性能 Tensor Core HGEMM
- 关键优化点（相比 mm_v6）:
  1. **mma.sync PTX (m16n8k16)** 替代 WMMA API，更精细的寄存器控制
  2. **cp.async** 异步拷贝 global→shared，不经过寄存器
  3. **双缓冲 (double buffering)** 计算与内存加载重叠
  4. **更大 block tile (128×256×32)** 更好的数据复用
  5. **ldmatrix** 指令高效加载 shared→register
  6. **FP32 累加器** 提高精度后转回 FP16 写出
  7. **Shared memory padding** (+8 half) 消除 bank conflict
- Tile 配置: BM=128, BN=256, BK=32, WM=64, WN=64
- 8 warps (2×4), 256 threads per block
- 更新了 mm.h（声明）和 main.cu（check + benchmark 集成）
- 目标架构: sm_80+ (Ampere/Hopper), 项目配置为 sm_90 (H100)

## mm_v7 精度修复
- 首版 mm_v7 精度严重不对，修复了以下关键 bug：
  1. **A 的 ldmatrix.x4 地址映射错误**: 原先用 `lane_id%16` 和 `(lane_id/16)*8` 作为行列，
     但 ldmatrix.x4 的 4 个 group(lane/8=0,1,2,3) 对应的行列应该是：
     `row = (lane_id%8) + ((lane_id/8)&1)*8`, `col = (lane_id/16)*8`
  2. **B 必须用 ldmatrix.x2.trans**: B 在 smem 中是 row-major [K][N]，而 mma.m16n8k16.row.col
     需要 B 是 col-major。必须用 `ldmatrix.sync.aligned.m8n8.x2.trans` 来做 8×8 块转置。
     原先错误使用了非转置版本 `ldmatrix.x2`。
  3. **cp.async 边界处理**: 当 pred=false 时（超出矩阵边界），需要清零 smem，
     改为手动 uint32_t 清零方式确保边界外数据为零。
